# Contributing to Polar

## Setup

```bash
go test ./...
```

Use the repository's normal Go workflow for builds and tests.

## Expectations

- Keep REST and MCP behavior aligned through the shared service layer.
- Update `README.md` for user-facing API or operational changes.
- Update `BLINK.md` when build, deploy, or verification behavior changes.
- Update `CHANGELOG.md` for notable repo changes.

Active unfinished work belongs in the workspace root `ROADMAP.md`.
